- 输出支持：
  - `real_cmi`

  - `cmi_rel_error`

在 `pipeline/run_external_baselines.py` 中新增 `_clip_target_distribution()` 函数，将 CTGAN 合成数据的目标列裁剪到参考训练集的 p1~p99 分位数范围，解决 R2 为负的主因（标签分布偏差）。

```python
def _clip_target_distribution(syn, ref, target_col, lo=0.01, hi=0.99):
    ref_vals = pd.to_numeric(ref[target_col], errors="coerce").dropna()
    lo_val, hi_val = float(ref_vals.quantile(lo)), float(ref_vals.quantile(hi))
    syn = syn.copy()
    syn[target_col] = pd.to_numeric(syn[target_col], errors="coerce").clip(lo_val, hi_val)
    return syn
```

## Section 23: 基线选型澄清 + 迁移学习四维分析（2026-04-20）

### 23.1 TVAE 优于 CTGAN 的根因分析

**结论**：NYC 交通事故数据对 CTGAN 结构性不友好，TVAE 因编码器-解码器架构更适配此类数据。

**核心原因**：

| 维度 | CTGAN 弱点 | TVAE 优势 |
|------|-----------|----------|
| 训练稳定性 | GAN 对抗训练在高维稀疏类别特征下极易模式塌缩 | VAE 变分推断目标为 ELBO，收敛稳定 |
| 标签分布 | 条件生成器对有界整数（伤亡人数 0-5）分布估计偏差大（R2 为负的根因 #1） | 连续隐空间自然拟合有界整数边缘分布 |

| 样本质量 | 条件向量采样方式对低频类别生成量不足 | 后验采样保证各类别覆盖 |
| Clip 补丁 | 已对 CTGAN 加了 clip 修正，证明其本身存在数值越界问题 | 无需 clip |

**建议论文表述**：
> CTGAN 在 TSTR 回归任务中出现 R²<0（n=2000: −0.52，n=10000: −0.27），根因为模式塌缩导致的标签分布偏差（Section 20 诊断）。此结果表明基于对抗训练的生成模型在高维稀疏类别 + 有界整数数值混合分布下具有结构性不稳定，将其作为失败基线保留在对比表中，为方法选型提供参考。

## Section 23: 基线选型澄清 + 迁移学习四维分析（2026-04-20）

### 23.1 TVAE 优于 CTGAN 的根因分析

**结论**：NYC 交通事故数据对 CTGAN 结构性不友好，TVAE 因编码器-解码器架构更适配此类数据。

**核心原因**：

| 维度 | CTGAN 弱点 | TVAE 优势 |
|------|-----------|----------|
| 训练稳定性 | GAN 对抗训练在高维稀疏类别特征下极易模式塌缩 | VAE 变分推断目标为 ELBO，收敛稳定 |
| 标签分布 | 条件生成器对有界整数（伤亡人数 0-5）分布估计偏差大（R2 为负的根因 #1） | 连续隐空间自然拟合有界整数边缘分布 |
| 38 维类别列 | PAC-GAN 判别器在 155 维 one-hot 下梯度稀疏，难优化 | 编码维度压缩，OHE 后矩阵仍可被 VAE 有效编码 |
| 样本质量 | 条件向量采样方式对低频类别生成量不足 | 后验采样保证各类别覆盖 |
| Clip 补丁 | 已对 CTGAN 加了 clip 修正，证明其本身存在数值越界问题 | 无需 clip |

**建议论文表述**：
> CTGAN 在 TSTR 回归任务中出现 R²<0（n=2000: −0.52，n=10000: −0.27），根因为模式塌缩导致的标签分布偏差（Section 20 诊断）。该结果表明基于对抗训练的生成模型在高维稀疏类别 + 有界整数数值混合分布下具有结构性不稳定，将其作为失败基线保留在对比表中，为方法选型提供参考。

**待补充实验**：计算 2017 测试集 vs 2025 测试集关键特征的 JS 散度（CRASH_TIME_SIN、VEHICLE_TYPE、IS_MULTI_VEHICLE）。

---

#### 维度 2：数据量与标签可用性（Data Volume & Label Availability）

| 方面 | 2017 源域 | 2025 目标域 |
|------|----------|------------|
| 训练集 | 173,533 行（充足） | **无**（仅有测试集） |
| 测试集 | 43,384 行 | 5000 行 |
| 标签可用性 | 完整标签 | 测试集有标签（可评估，不可 fine-tune） |
| 合成数据 | 10,000 行（ours_full_model_full.csv） | 无 |

**模式：Zero-Shot Transfer（零样本迁移）**

- 无法对 2025 数据微调模型（无训练集）
- 迁移评测：2017 合成数据训练 → 2025 真实数据测试（跨时间 TSTR）
- 这是验证因果约束泛化能力的最严格场景

**可选增强**：如能获取少量 2025 年带标签训练数据（200-500 行），可增加 Few-Shot Adaptation 对比。

---

#### 维度 3：知识共享度（Knowledge Transferability）

| 知识类型 | 共享程度 | 说明 |
|---------|---------|------|
| 事故物理因果（超速/酒驾→严重性） | **高（~0.90）** | 物理规律跨年份稳定 |
| 道路基础设施特征（OSM：信号灯距离、车道数） | **高（~0.85）** | NYC 路网结构 2017-2025 变化小 |
| 时间周期性规律（早高峰/周五晚） | **中（~0.55）** | 疫情后通勤模式改变，需验证 |
| 行为驾驶模式（is_speeding/is_distracted） | **中低（~0.45）** | 电动车渗透、手机法规更严，标签分布可能漂移 |
| V2 因果 DAG 结构（222 边） | **中高（~0.70）** | 物理+空间边稳定，行为边需验证 |
| 模型参数直接迁移可行性 | **高** | HC-DiffTraffic 是生成模型，参数不随测试集变化 |

**核心论点**：
> 因果约束显式编码了物理和结构规律（维度 3 中共享度最高的那些边），使模型在 2017→2025 分布漂移时保持更好的因果一致性，而无约束模型（TabDDPM/TVAE）更可能过拟合 2017 年的行为分布。

---

#### 维度 4：迁移学习目标（Transfer Learning Objective）

**本项目迁移目标**：验证因果约束对跨时间分布漂移的鲁棒性（Causal Generalization）

| 目标 | 当前实现 | 状态 |
|------|---------|------|
| 跨年份分类准确率 | evaluate_postcovid_transfer.py：ours_full_model vs ablation_no_causal | 待 Stage3 完成 |
| 迁移退化率 | composite_score(2025) / composite_score(2017) - 1 | 待实现 |
| 结构层泛化（CMI 稳定性） | 2025 测试集上 CMI 误差 vs 2017 | 需启用 CMI 计算 |
| 跨时间 TSTR F1 | 2017合成→2025真实测试 F1-macro | **核心指标，优先计算** |

**推荐评测指标（迁移场景专用）**：

```text
cross_temporal_tstr_f1_macro     # 2017 合成 + 2017 真实训练，2025 真实测试
transfer_degradation_rate        # (F1_2017 - F1_2025) / F1_2017，越小越好
causal_stability_index           # CMI偏差(2025) / CMI偏差(2017)，<1 为改善
```
