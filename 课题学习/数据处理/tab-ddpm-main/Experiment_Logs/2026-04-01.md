## 2. 问题汇总与根因定位

### 2.1 问题 A: postprocess 阶段整数转换崩溃

- 现象: `IntCastingNaNError: Cannot convert non-finite values (NA or inf) to integer`
- 场景: `ablation_no_hierarchy` full 采样后处理阶段
- 根因: 分类/目标列存在 `NaN/inf`，直接 `astype(int)` 触发异常

### 2.2 问题 B: full 训练中 checkpoint 清理崩溃

- 现象: `PermissionError: [WinError 32]`，报错栈指向 `os.remove(f)`
- 场景: `ablation_no_causal` stage3 full 训练中，删除 `best_ema_model_*.pt` 时
- 根因: Windows 文件锁导致瞬时删除失败
- 说明: 用户贴出的 `2026-03-30T17:19:15+08:00` 栈属于旧轮次失败，不是新轮次再次触发

### 2.3 问题 C: rerun 脚本偶发路径解析错误

- 现象: `can't open file ...\pipeline\run_all_experiments.py`
- 场景: 从非项目 cwd 启动脚本时相对路径失效
- 根因: 启动目录与脚本相对路径耦合

---

## 3. 修复与加固

### 3.1 `src/postprocess_samples.py` 修复

- 对分类列和目标列统一进行:
  - `to_numeric(errors='coerce')`
  - `inf/-inf -> NaN`
  - 合理 `fillna` 后再安全转换
- 结果: 之前失败样本可成功后处理落盘

### 3.2 `src/train_hierarchical.py` 修复

- 新增 `_safe_remove_file(path, retries, delay_sec)`
- 在 best/best_ema checkpoint 清理处，用重试删除替代直接 `os.remove(f)`
- 结果: 避免 Windows 短暂锁文件导致训练硬中断

### 3.3 `pipeline/rerun_full_after_fix.ps1` 加固

- 固定使用 `$PSScriptRoot` 推导项目根
- 新增 `python` 与 `run_all_experiments.py` 存在性预检
- 执行改为绝对脚本路径
- 结果: 避免从错误 cwd 启动导致的假失败
