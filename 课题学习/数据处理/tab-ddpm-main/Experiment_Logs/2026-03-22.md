## 附录B: 评估指标定义与解释

### B.1 规则与一致性指标
| 指标 | 定义 | 趋势解释 |
|---|---|---|
| `logic_violation_rate` | 违反硬逻辑约束的样本占比 | 越低越好，0表示硬规则一致性最佳 |
| `Commonsense_Violation_Rate` | 违反常识规则的样本占比 | 越低越好 |
| `Correction_Rate` | 采样后被规则修正的比例 | 中性指标；高说明原始采样违规多，低通常更稳 |

### B.2 分布保真指标
| 指标 | 定义 | 趋势解释 |
|---|---|---|
| `numeric_drift_avg` | 数值字段均值/方差漂移的平均量 | 越低越好 |
| `categorical_tv_avg` | 类别字段总变差距离（TV）的平均值 | 越低越好 |
| `avg_js_divergence` | 多个类别分布 JS 散度平均 | 越低越好 |
| `y_wasserstein` | 目标变量分布 Wasserstein 距离 | 越低越好 |

### B.3 下游任务效用指标
| 指标 | 定义 | 趋势解释 |
|---|---|---|
| `TSTR_F1_Macro` | Train on Synthetic, Test on Real 的宏平均F1 | 越高越好，衡量生成数据训练价值 |
| `fidelity_ratio` | 稀疏目标上 TSTR 与 real-train 的相对比值 | 接近1更好，低于1表示效用损失 |
| `test_rmse` | 回归误差均方根 | 越低越好 |
| `test_r2` | 回归拟合优度 | 越高越好（负值通常表示较差） |
| `cause_weighted_f1` | 事故主因加权F1 | 越高越好 |
| `injury_auc` | 受伤二分类AUC | 越高越好 |

### B.4 隐私风险指标
| 指标 | 定义 | 趋势解释 |
|---|---|---|
| `dcr_mean` | 生成样本到最近真实样本的距离均值 | 过低可能表示贴近过度；需结合任务指标看 |
| `dcr_exact_match_ratio` | 与真实样本完全匹配的比例 | 越低越安全，非零需重点关注 |

### E.3 CausalDiffTab 风格训练机制改造
针对“前 20 个 epoch 因果约束过强导致收敛困难”的问题，在训练脚本中引入两项机制。

修改文件：
- `train_causal_yandex.py`
- 新增兼容入口 `train_causal.py`

关键改动：
1) 张量级因果掩码构建：
- 新增 `build_tensor_penalty_mask(column_info, causal_edges)`。
- 支持按 one-hot 展开后的总维度构建 `D x D` 掩码。
- 同特征内部与有因果边特征对置 `0`，其余为 `1`。

2) 外积因果惩罚：
- 新增 `causal_outer_product_penalty(eps_hat, penalty_mask)`。
- 对 `eps_hat [B, D]` 做外积并 batch 平均，使用 Frobenius 范数平方作为 `L_causal`。

3) 混合自适应权重：
- 新增 `compute_hybrid_causal_weight(...)`。
- 实现：`w_hybrid = w_max * 0.5 * (exp(-|ΔL|) + 1/(1+sigma_approx))`。
- 通过 `sigma_approx` 的早期线性衰减，自动抑制训练初期的约束强度。

4) 总损失更新：
- `L_total = L_base + w_hybrid * L_causal`。
- 训练日志新增记录：`base_loss, causal_loss, hybrid_weight, delta_l, sigma_approx`。
