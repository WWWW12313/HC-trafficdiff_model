## 1. 项目架构确立：Hierarchical CausalDiffTab

### 1.1 核心理念
放弃"全量变量同时生成"的传统表格扩散策略，改为**分层生成 (Hierarchical Generation) + 离线知识增强**：
1. **Stage 1 - 空间生成**：扩散模型先生成经纬度锚点 (LATITUDE, LONGITUDE)
2. **Stage 2 - 离线物理映射**：根据生成经纬度 + 基准时间，从本地 OSM/天气数据匹配路况、限速、天气
3. **Stage 3 - 约束生成**：以 Stage 1+2 为条件，结合 NOTEARS 因果图做条件去噪，生成事故时间与类型

### 1.2 工作目录
主开发目录迁移至 `CausalDiffTab-main/CausalDiffTab-main/`，以下简称 CDT 根目录。

## 3. Task 2: NOTEARS 因果图挖掘脚本

**新增文件**: `CDT/src/causal_discovery_notears.py`

### 3.1 变更目的
用 NOTEARS 算法在预处理后的混合类型数据上学习因果 DAG，输出 N×N 邻接矩阵。

### 3.2 核心功能

| 模块 | 说明 |
|------|------|
| `load_preprocessed_data()` | 从 npy 或 CSV 加载预处理数据 |
| `prepare_for_notears()` | 子采样(默认 20k)+ StandardScaler + NaN 处理 |
| `run_notears_original()` | 调用 CausalDiffTab 自带 notears 库（linear/MLP） |
| `run_notears_causallearn()` | causal-learn 库后端（备选） |
| `run_notears_scipy()` | 纯 scipy 实现（最终 fallback，无外部依赖） |
| `_notears_linear_scipy()` | NOTEARS Linear 的完整增广拉格朗日 scipy 实现 |
| `summarize_dag()` | DAG 统计摘要：边数、密度、根节点、top-K 边 |

### 3.3 三层后端容错
1. **original** (默认): CausalDiffTab 自带 `notears.linear` / `notears.nonlinear`
2. **causallearn**: causal-learn 库
3. **scipy**: 纯 numpy/scipy 实现，无第三方因果库依赖

### 3.4 输出
- `configs/causal_matrix_notears.npy` — 二值化邻接矩阵 (N×N, int32)
- `configs/causal_matrix_notears_weights.npy` — 连续权重矩阵 (N×N, float64)
- `configs/causal_matrix_notears.json` — DAG 摘要 (edges, density, roots, config)

## 12. 架构优化 Round 4: Full 训练前置优化 (动态参数调整与自适应正则化)

基于 balanced (200 epoch) 训练发现 loss 在 100 epoch 后进入平台期并出现验证集反弹。
在正式 full (160k x 4000 epoch) 训练前，引入四大优化机制。

### 12.1 Task 1: CosineAnnealingLR + Early Stopping

**文件**: `src/train_hierarchical.py`

- **CosineAnnealingLR**: 替换 ReduceLROnPlateau
  - `T_max = total_epochs`, `eta_min = 1e-6`
  - 学习率从初始值平滑余弦衰减至 1e-6，避免平台期震荡
  - 每个 epoch 结束调用 `scheduler.step()` (无需传 loss)
- **Early Stopping**: `patience = 200`
  - 连续 200 epoch Eval Total Loss 未创新低时终止训练
  - 自动保存 Best Model，避免过拟合浪费算力

### 12.2 Task 2: Causal Penalty Linear Warmup

**文件**: `tabdiff/models/unified_ctime_diffusion.py`, `src/train_hierarchical.py`

- **warmup_epochs** = `int(total_epochs * 0.2)` (full: 800 epochs)
- **公式**: `causal_lambda = causal_weight_max * min(1.0, current_epoch / warmup_epochs)`
- 前 20% 训练时间因果正则化权重从 0 线性增长到 1.0
- 防止训练初期 DAG 惩罚破坏底层联合分布拟合

### 12.3 Task 3: QuantileTransformer

**文件**: `src/data_processor.py`

- **替换**: StandardScaler → `QuantileTransformer(output_distribution='normal', n_quantiles=1000)`
- **原因**: EDM 扩散模型的去噪先验是标准正态分布，强行拟合非高斯/长尾连续数据导致 CLoss 降不下去
- QuantileTransformer 强制映射为精确的 N(0,1)，与扩散先验完美对齐
- **逆变换**: scaler 以 pickle 保存至 `data/processed/continuous_scaler.pkl`
- CLI 默认从 `--norm_method standard` 改为 `--norm_method quantile`

### 12.4 Task 4: Dropout 正则化

**文件**: `tabdiff/modules/main_modules.py`, `src/train_hierarchical.py`

- **MLPDiffusion**: 每个 SiLU 激活后插入 `nn.Dropout(p=0.15)`
